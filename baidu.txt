查看分支： git branch
创建分支： git branch 分支名称
	删除分支：git branch -d 分支名称
切换分支： git checkout 分支名称
分支合并： git merge 要合并得分支
	注意点：切换分支在合并，注意合并得所属关系，例如 B合并到A分支下，那么首先得切换至A分支后，执行git merge B
	合并过程中可能会产生冲突： 基础办法是手动进行修改
工作流概念：
	

2.6 
	1.给远程仓库起别名
	git  remote add origin 远程仓库地址 
	2.向远程推送代码
	git push -u origin 分支
	

	1.克隆远程仓库代码
	git clone 远程仓库的地址（内部已经实现仓库地址别名）
	2.切换分支
	git checkout 分支名称
	

	在公司进行开发
	1 切换至dev 分支进行开发
	git checkout dev
	2 把master分支合并到dev（更新最新得代码,只执行一次）
	git merge mater
	3 修改代码
	4 提交代码
		git add .
		git commit -m "xxx"
		git push origin dev
		
	回到家中继续写代码
		1 切换到dev分支进行开发
		git checkout dev
		2 拉代码
		git pull origin dev
		3 继续开发
		4 提交代码
		git add .
                git commit -m "xxx"
                git push origin dev

	开发完毕，上线操作。
		1.将dev分支合并到master，运行上线
			git checkout master
			git merge dev
			git push origin master
		2.把dev分支也推送到远程
			git checkout dev
			git merge master
			git push origin dev
